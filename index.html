  1. 环境配置

本人电脑是Windows 10专业版64位

  1. 安装Node.js
  • 进入Node.js官网下载安装包, 下载地址link,建议安装长期支持版本也就是LTS, 毕竟会经常维护和支持, 稳定性也更好
  • 配置好Node.js的环境配置
  • 检查是否安装成功可以在cmd中运行node -v和npm -v
  1. 安装Git
  • 进入官网Git官网下载安装包,下载地址link, 刚开始接触建议默认安装模式
  • 记得配置好环境变量, 在桌面任何空白位置鼠标右键如出现Git GUI和Git Bash以下选择及安装成功
  1. GitHub账号注册和新建项目
  • 新建项目安装固定格式: 账户名.github.io, 然后勾选Initialize this repository with a README.
  • 设置成功即可访问你的blog, 访问地址: https://账户名.github.io
  1. 安装Hexo
  • 建议在非系统盘下建一个文件夹, 比如我在F盘建了blog文件夹
  • 在此目录运行命令安装Hexonpm install hexo -g
  • 运行命令hexo -v, 检查hexo看是否安装成功
  • 运行命令hexo init, 初始化该文件夹
  • 运行命令npm install,安装所需要的组件
  • 运行命令hexo g
  • 运行命令hexo s, 在本地启动服务即可访问改地址
  1. 设置Git
  • 设置Git的user.name和user.email
  • 运行命令ssh-keygen -t rsa -C "1559764912@qq.com", 默认选择连续三个回车, 将会生成密钥, 得到两个文件id_rsa和id_rsa.pub, 默认文件地址: C:\Users\Administrator\.ssh
  • 运行eval "$(ssh-agent -s)", 添加秘钥到shh-agent
  • 运行命令ssh-add ~/.ssh/id_rsa, 添加生成的ssh key到ssh-agent
  1. 设置GitHub
  • GitHub账号设置中添加ssh
  • 新建一个ssh key, 将id_rsa.pub文件内容粘贴至key中
  • Git中运行运行命令ssh -T git@github.com, 测试添加ssh是否成功, 如果看到你的用户名即表示已成功
  • 在blog文件夹中找到_config.yml文件,修改其值, 替换成你的用户名
  1. 发布博客
  • 在cmd中运行hexo new post "博客名", 此时会在source > _posts文件夹中创建该文件,以markdown格式保存, 以后可以把文件放在此目录下
  • 运行命令npm install hexo-deployer-git --save, 安装此扩展以便生产和部署文章
  • 生产部署文章hexo d -g
  • 此时即可访问你个人博客https://用户名.github.io
